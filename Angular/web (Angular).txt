Data Binding in Angular
1. Introduction

Data binding in Angular is the process of connecting application data (TypeScript code) with the view (HTML template).
It allows automatic synchronization of data between the component and the UI.

2. Types of Data Binding in Angular (4)

1. Interpolation (One-Way Data Binding â†’ TS â†’ HTML)

Used to display component data in the HTML.
Syntax:

{{ variableName }}


Example:
ts
name = 'Akhi';

html
<p>Hello {{ name }}</p>
==========================================
2. Property Binding (One-Way â†’ TS â†’ HTML Attribute/Property)

Used to bind data to HTML element properties, not attributes.

Syntax:
[elementProperty]="value"


Example:

<input [value]="name">

=========================================
3. Event Binding (One-Way â†’ HTML Event â†’ TS function)

Used to call TypeScript functions when events occur.

Syntax:

(eventName)="functionName()"


Example:

<button (click)="showMessage()">Click Me</button>

=======================================
4. Two-Way Data Binding (Two-Way â†’ TS â†” HTML)

Achieved using ngModel.

Syntax:

[(ngModel)]="variableName"


Example:

<input [(ngModel)]="username">
<p>You typed: {{ username }}</p>

Form validation 

Angular Forms is a powerful module in Angular that allows developers to create dynamic, interactive, and validated forms in web applications. Forms are essential for collecting user input, such as registration details, login credentials, or any data submission. Angular provides built-in directives and classes to manage form data, validation, and user interactions efficiently.

There are two main ways to implement forms in Angular:

a) Template-Driven Forms

Form logic is defined mostly in the HTML template.

Uses directives like ngModel, required, minlength, etc.

Simpler to implement, suitable for small forms.

Example:

<form #userForm="ngForm" (ngSubmit)="submitForm(userForm.value)">
  <input type="text" name="username" ngModel required>
</form>

============================================
b) Reactive (Model-Driven) Forms

Form logic is defined in the component TypeScript code.

Uses FormGroup, FormControl, and Validators.

Suitable for large, complex forms.

Provides better scalability and testability.

Example:

this.userForm = new FormGroup({
  username: new FormControl('', Validators.required)
});

<form [formGroup]="userForm" (ngSubmit)="submitForm()">
  <input formControlName="username">
</form>

3. Types of Angular Forms

Angular provides two types of forms:

(Template-Driven Forms)

Easy to use.

Two-way data binding with ngModel.
Validation done via directives in HTML.

(Reactive Forms)

Form model defined in TypeScript.

More powerful, flexible, and scalable.

Allows dynamic form controls and custom validators.

============================================
4. Validations in Angular Forms

Angular provides built-in validators to ensure data integrity and correctness:

Built-in Validators

required â†’ ensures a field is not empty.

minLength / maxLength â†’ ensures correct string length.

pattern â†’ checks input against a regex pattern.

email â†’ validates email format.

min / max â†’ validates numerical values.

Custom Validators

Developers can create custom validation logic to meet specific requirements.

Example: password strength, age validation, username availability.

Validation Feedback

Angular provides real-time validation feedback to users via invalid, touched, and dirty states.

========================================
5. Why Use Angular Forms?

Ease of Data Binding: Connects form controls to model objects easily.

Validation: Built-in validators and custom validators for robust checks.

Reactive Updates: Form values and states are updated automatically.

Scalability: Reactive forms are suitable for complex enterprise applications.

Testability: Easier to write unit tests for form logic and validation.

Dynamic Forms: Can add or remove controls at runtime (Reactive Forms).
