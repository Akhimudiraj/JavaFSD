1. What is JWT?

JWT (JSON Web Token) is a secure, compact token used to represent user identity.
It is commonly used for stateless authentication in modern web apps.

A JWT contains 3 parts:

Header ‚Äì algorithm & token type
Payload ‚Äì user info (e.g., username, roles)
Signature ‚Äì verifies that the token is not tampered with

JWT Authentication ‚Äî Simple Explanation (Like a Story)

Imagine you want to enter a secure building (your application).

‚≠ê Step 1: User knocks on the door

You send username & password to /login.

Meaning:
You are asking the server ‚Üí ‚ÄúPlease let me in.‚Äù

‚≠ê Step 2: Security guard checks your ID

Spring Security checks:

Is the username correct?

Is the password correct?

Meaning:
Server verifies your identity.

‚≠ê Step 3: Guard gives you a badge (JWT Token)

If you're valid, the server gives you a token.

This token has:

Your name

Your role (USER/ADMIN)

Expiry time

A signature so it cannot be copied

Meaning:
Token = Entry pass.

‚≠ê Step 4: You keep the badge

You store the token on your side (browser or mobile).

Important:
The server does NOT keep any record of the token ‚Üí That‚Äôs why it's called stateless.

‚≠ê Step 5: Every time you enter a new room, you show the badge

When you call any API:

GET /products
Authorization: Bearer <token>


Meaning:
You show your badge to enter another room.

‚≠ê Step 6: A scanner checks your badge (JWT Filter)

Before entering the room, a machine (filter) checks:

Is your token still valid?

Is the signature correct?

Has it expired?

Does your role allow entry?

Meaning:
Spring Security checks the token BEFORE processing your request.

‚≠ê Step 7: If badge is valid ‚Üí you get access

After verification:

Spring marks you as authenticated

You can now access the endpoint

Meaning:
You‚Äôre allowed to do what your role permits.

‚≠ê Step 8: If badge is invalid ‚Üí Access denied

If token is:

expired

fake

tampered

missing

Then Spring Security returns:

401 Unauthorized

Login ‚Üí Get token ‚Üí Send token on every request ‚Üí Filter checks token ‚Üí Access granted.
