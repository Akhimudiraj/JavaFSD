Dependency Injection is a design pattern in which the IoC container (Spring) is responsible for providing the dependencies (objects) that a class needs, instead of the class creating them itself.


in simple 

Employee e = new Employee();
Department d = new Department();
e.setDepartment(d);

In Spring, you don’t create the object manually.
You tell Spring in the configuration what dependencies are needed —
Spring then injects (supplies) them automatically.

That’s called Dependency Injection.

so in setter 
it injected using setter methods 
like setter program

package spring;

public class Employee {
    private int id;
    private String name;

    public void setId(int id) { this.id = id; }
    public void setName(String name) { this.name = name; }

    public void display() {
        System.out.println("ID: " + id + ", Name: " + name);
    }
}
